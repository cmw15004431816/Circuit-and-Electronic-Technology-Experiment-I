实验原理

首先明确本实验中用到的两种芯片：与非门74HC00和规模集成计数器74LS161，它们的引脚排列如下：

74LS161芯片的11-14号引脚（分别记为\(Q_A、Q_B、Q_C、Q_D\)）为输出，其输出为当前计数的二进制码，11号引脚为最高位（the most significant bit, MSB），14号引脚为最低位（the least significant bit, LSB），11（\(Q_A\)）、12（\(Q_B\)）、13（\(Q_C\)）、14（\(Q_D\)）号引脚分别表示从高到低的4位输出二进制码。此外还需要注意的是（图中已经标出），74LS161芯片的时钟是上边沿触发，清零则是下边沿触发。

除了上述两种芯片以外，我们还用到了七段数码管作为可视化输出。每个七段数码管接受四个输入D、C、B、A，管内置电路将上四个输入经过8-4-2-1码方式静态译码后在管上显示对应的数字（D对应高位8，C对应4，B对应2，A对应1，如输入1001数码管显示5）。

注意计数器的输出是二进制码，七段数码管的输入为8-4-2-1码，两者仅在0-9范围内相同。如果你将计数器输出的大于9的结果接入数码管的输入，由于此时的二进制码是无效的8-4-2-1码，故七段数码管将熄灭（如输入1101数码管将熄灭）。

有了上面的理论基础，现在我们可以开始讨论24进制计数器和60进制计数器了。

无论是哪种计数器，它都由一个个位和一个十位构成，且个位都是一个十进制计数器。对于24进制计数器，计满24后个位十位全部清零；对于60进制计数器，计满60后个位十位全部清零。

下面先讨论60进制计数器。

先看个位，它是一个十进制计数器，逢十进一。而个位每进一，十位就要计一，同时个位清零；十位相当于是一个6进制计数器，计到6时就将十位清零。

由于十位的计数只能由个位的进位所导致，而个位每次发生进位，都会将自身清零，所以对于60进制计数器，当十位计到6的时候，只需要对十位本身进行清零即可，不需要对个位进行处理，因为个位一定已经被自己清零了。这一点和下面将要讨论的24进制计数器不同。

最后总结一下60进制计数器的实现方法：用到个位和十位两个74LS161芯片，个位的时钟引脚接实验箱时钟信号，每当个位输出变成10的时候，给自己的清除引脚一个下边沿实现清零，同时给十位的时钟引脚一个上边沿，触发十位计数一次。当十位的输出变成6的时候，给十位的清除引脚一个下边沿实现十位清零。个位和十位的输出分别接到七段数码管上。至此完成了用七段数码管输出的60进制计数器。

以60进制计数器为基础，让我们来讨论24进制计数器。

个位仍然是一个十进制计数器，但十位不同（绝不能想当然地认为是相当于一个二进制计数器）。个位逢十进一，自身清零。除此之外，当十位计到2，并且个位计到4的时候，对个位和十位同时清零。你会发现，个位的清零有两种情况（自身满10或当前计到24），十位的清零有一种情况（当前计到24），并且，当前计到24的清零，它并不仅仅依赖于单独的个位或者十位，而是由个位和十位共同决定。这使得24进制计数器要比60进制计数器稍显复杂。但是，理清了上述清零的情况以后，我们也就可以来总结24进制计数器的实现方法了。

24进制计数器的实现方法总结：个位和十位使用的芯片与60进制计数器相同。个位的时钟引脚依然接实验箱的时钟信号，个位的清除引脚接以下两种情况的或：个位输出变成10，或者，十位输出2，并且同时个位输出4——只要这两种情况至少发生其一（实际上，根据我们的实现逻辑，它们可能且只可能发生其一，所以这里使用或逻辑与使用异或逻辑理论上应该能得到相同的实现结果，但是本次实验中我们采用或逻辑，原因见下文），我们就给个位的清除引脚一个下边沿（注意，最终给出这个下边沿，是靠一个与门或者一个同或（异或非）门来实现的，详见下文。上面所说的是逻辑，并不是具体实现时真正使用的门的种类）。每当个位输出变成10的时候。给十位的时钟引脚一个上边沿，触发十位计数一次。当十位输出2，并且同时个位输出4的时候，给十位的清除引脚一个下边沿实现十位清零（当然，如前文所述，此时也给了个位的清零引脚一个下边沿实现了个位清零）。个位和十位的输出分别接到七段数码管上。至此就完成了用七段数码管输出的24进制计数器。

上文中，我们一直在说“给”一个引脚一个下边沿或者上边沿。那怎么“给”这个上/下边沿呢？我们使用与非门74HC00实现。首先来说下边沿，比如，当个位输出10的时候，我们想给出一个下边沿，那么我们就把\(Q_B\)和\(Q_D\)作为与非门的两个输入，则该与非门的输出将在——\(Q_B\)和\(Q_D\)都变成1，对应到计数器，就是当该计数器（的个位）从9计到10——的那一刹那由1变为0，即给出了一个下边沿。那么怎么给出上边沿呢？一个十分简单的思路当然是使用非门和与非门实现，不改变上述与非门的输入，只需要将与非门的结果通过非门，产生的下边沿就变成了上边沿。但是，这种实现并没有被本次实验所采用，原因如下：首先，本实验不允许使用除了与非门之外的其他任何逻辑门，尽管事实上我们有（这也决定了，我们上文提到的或逻辑也必须使用与非门实现，详见下文），此外，我们本次实验使用的是异步时钟（asynchronous clock），尽管与非门是功能完备（functional complete）门，我们可以用与非门实现任何基本逻辑门（basic logic gates），当然也包括非门，但这可能会增加门输入代价（gate cost），并会使我们的电路更加复杂且低效，而且加入非门带来的门延时（gate delay）将可能导致结果错误。我们完全可以修改与非门的输入，从而在相应时刻产生我们需要的上边沿。仍然考虑当个位输出10的时候，此时我们想给出一个上边沿。注意到，在我们本次实现的计数器中，上边沿和下边沿总是成对出现（因为一个信号发生改变之后总还要恢复到原来的值），所以我们可以考虑提前产生一个下边沿并将之舍弃，然后接收信号恢复时的上边沿。所以，我们此时可以把\(Q_A\)和\(Q_D\)作为与非门的两个输入，则该与非门的输出将在——\(Q_A\)和\(Q_D\)都变成1，对应到计数器，就是当该计数器（的个位）从8计到9——的那一刹那由1变为0，即给出了一个下边沿（和上文一样）。但我们并不关心下边沿，于是考虑下一步计数：与非门的输出将在——该计数器（的个位）从9计到10，也就是\(Q_D\)保持1，\(Q_A\)从1变成0——的那一刹那由0变为1，即在计数器输出10的时候给出了一个上边沿。这样，仅仅是改变了电路的连接方式，我们就实现了产生上升沿，并且不会有门延时和门输入代价的问题。

上面我们已经说过，要使用与非门实现与门，那么如何实现呢？这需要一点点的布尔代数（Boolean algebra）知识。我不会在这里讲布尔代数，直接给出结果如下：

\[ A \text{ AND } B = (A \text{ NAND } B) \text{ NAND } (A \text{ NAND } B) \]

上文还说过，使用同或（异或非）门和使用与门将产生同样结果，让我们考虑用与非门实现同或门：

\[ A \text{ XNOR } B = ((A \text{ NAND } (A \text{ NAND } B)) \text{ NAND } \\ (B \text{ NAND } (A \text{ NAND } B))) \text{ NAND } ((A \text{ NAND } (A \text{ NAND } B)) \text{ NAND } \\ (B \text{ NAND } (A \text{ NAND } B))) \]

两种门的实现的复杂程度高下立判。同或门的电路级数高，门输入延迟也更高。我们没有必要放弃简单的与门不用，非要去用更加复杂的同或门。至此就解释了，我们的实验中为什么用的是或逻辑（与门）而不是异或逻辑（同或门）。

对于只有1位的信号（比如我们本实验中的信号），我们可以将之与1(逻辑高电平)与非，得到它的反，也就是说，我们可以这样，实际上在本实验中也就是这样实现与门（下面的A、B只能是一位的信号）：

\[ A \text{ AND } B = (A \text{ NAND } B) \text{ NAND } 1_{logic} \]

到此为止，本次实验的原理就已经阐释的十分清楚了。
